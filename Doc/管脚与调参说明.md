# 管脚与调参说明

## 一、硬件管脚

| 模块 | STM32 管脚 | 说明 |
|---|---|---|
| HC-SR04 TRIG | PB14 | 前方超声波触发 |
| HC-SR04 ECHO | PB15 | 前方超声波回波；PB15 为 FT 数字输入，可选分压保护 |
| 左红外 | PA12 | 检测到障碍物为低电平 |
| 右红外 | PA11 | 检测到障碍物为低电平 |
| USART1 TX | PA9 | 接 USB-TTL 的 RX |
| USART1 RX | PA10 | 接 USB-TTL 的 TX，当前预留接收 |
| L298N IN1 | PB6 | 左电机方向控制 |
| L298N IN2 | PB7 | 左电机方向控制 |
| L298N IN3 | PB8 | 右电机方向控制 |
| L298N IN4 | PB9 | 右电机方向控制 |
| L298N ENB | PA1 / TIM2_CH2 | 右电机 PWM |
| L298N ENA | PA2 / TIM2_CH3 | 左电机 PWM |

## 二、红外电平

本工程已经设置为：检测到障碍物 = 低电平。

```c
#define IR_BLOCKED_LEVEL 0
```

代码位置：`USER/config.h`。

## 三、PWM 调节

所有 PWM 值均在 `USER/config.h` 中集中修改。本版已把差速驱动 PWM 调低：

```c
#define MOTOR_FORWARD_PWM         750
#define MOTOR_SLOW_PWM            620
#define MOTOR_TURN_DELAY_PWM      620
#define MOTOR_TURN_INNER_PWM      520
#define MOTOR_TURN_OUTER_PWM      780
#define MOTOR_TURN_BACK_SPIN_PWM  820
```

本版左右 90°转弯使用低速脉冲式差速转弯，左转时右轮快、左轮慢；右转时左轮快、右轮慢。死路掉头使用一侧正转、一侧反转的原地左旋掉头，更容易完成 180°。

## 四、转弯时间调节

```c
#define MAZE_TURN_DELAY_FORWARD_MS 220
#define MAZE_TURN_STEP_MS         120
#define MAZE_TURN_STEP_PAUSE_MS   20
#define MAZE_TURN_90_MS           720
#define MAZE_TURN_BACK_MS         1500
```

如果 90°转弯不足，增大 `MAZE_TURN_90_MS` 或略微提高 `MOTOR_TURN_OUTER_PWM`；如果转过头，减小该值。想让动作更细腻，可减小 `MAZE_TURN_STEP_MS` 并适当增加总时间。掉头由 `MOTOR_TURN_BACK_SPIN_PWM` 和 `MAZE_TURN_BACK_MS` 调节。

检测到需要转弯后，如果前方距离安全，小车会先用 `MOTOR_TURN_DELAY_PWM` 低速前进 `MAZE_TURN_DELAY_FORWARD_MS`，让车身更深入路口后再开始转弯。

转弯结束后会锁定触发转弯的一侧。例如右侧空旷触发右转后，会锁定右侧；锁定期间不响应另一侧红外，也不会恢复普通直行决策，而是继续朝锁定侧做小段差速修正，直到右侧红外重新检测到墙/障碍后才解锁。

## 五、算法说明

迷宫算法位于 `APP/maze.c`。

核心逻辑已改为左手原则：

```text
左侧无障碍 -> 左转
前方安全 -> 前进
右侧无障碍 -> 右转
否则 -> 原地左旋掉头
```

## 六、USART1 串口状态输出

- PA9：USART1_TX，接 USB-TTL 的 RX。
- PA10：USART1_RX，接 USB-TTL 的 TX，当前预留接收。
- 串口参数：115200-8-N-1。
- 输出格式：`US=18cm,US_OK=1,IR_L=BLOCK,IR_R=CLEAR,PWM_L=+600,PWM_R=+800`。
- `US_OK=0` 表示超声波本次测距失败，车辆会停止并等待下一次检测。
- `IR_L` 和 `IR_R` 分别表示左右红外状态：`BLOCK` 为检测到障碍物，`CLEAR` 为无障碍。
- `PWM_L` 和 `PWM_R` 分别表示左右电机实际 PWM。正数为前进，负数为后退，`0` 为停止。
- 输出周期：约 0.3 s。
- 开关、波特率和输出周期在 `USER/config.h` 中调整：

```c
#define UART_STATUS_ENABLE        1
#define UART1_BAUDRATE            115200U
#define UART_STATUS_INTERVAL_MS   300U
```

注意：USB-TTL 与 STM32 必须 GND 共地。

## 七、HC-SR04 电平保护

HC-SR04 通常使用 5V 供电，其 ECHO 输出约为 5V。STM32F103 的 PB15 属于 FT（5V tolerant）数字输入，因此可以直接连接。若使用来源不明的兼容板或希望增加保护，也可以增加分压电阻或电平转换模块。
